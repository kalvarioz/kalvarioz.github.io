Overview

Some people think that Linux is less secure because it doesn't have the sleek and persistent features that Windows has with antivirus software and Windows Defender. However, Linux takes a fundamentally different approach to security that emphasizes prevention over detection.

Linux security consists of regular patching, strict user permissions (least privilege principle), firewalls (UFW/iptables), SSH hardening, and mandatory access controls (SELinux/AppArmor). Key measures also include disabling unused services, using encryption, strong authentication (2FA/SSH keys), and continuous monitoring and logging to minimize the attack surface.

This page covers fundamental security tools and how you can use them to set up a protected desktop environment in Linux.

Firewall - UFW (Uncomplicated Firewall)

UFW (Uncomplicated Firewall) is the default, user-friendly command-line firewall configuration tool for Ubuntu and Debian-based Linux systems. It acts as a frontend for iptables or nftables to simplify managing network traffic (IPv4/IPv6).

Basic UFW Commands:


                        # Enable the firewall
                        sudo ufw enable

                        # Check status
                        sudo ufw status verbose

                        # Allow SSH
                        sudo ufw allow 22/tcp

                        # Allow HTTP and HTTPS
                        sudo ufw allow 80/tcp
                        sudo ufw allow 443/tcp

                        # Deny a specific port
                        sudo ufw deny 23/tcp

                        # Default policies
                        sudo ufw default deny incoming
                        sudo ufw default allow outgoing

Why does it matter?

Reduces Attack Surface: By blocking unnecessary services and ports, fewer entry points exist for attackers to exploit.
Prevents Unauthorized Access: Effectively stops unauthorized connection attempts to services running on your system.
Accessibility: Makes robust firewall protection accessible to beginners and pros alike, ensuring firewalls are actually used instead of being too complex to configure.

AppArmor - Mandatory Access Control

AppArmor (Application Armor) is a Linux security module that implements Mandatory Access Control (MAC) by confining programs to a limited set of resources using per-program profiles. Unlike traditional Discretionary Access Control (DAC), AppArmor policies define what files, network access, and capabilities an application can use, preventing unauthorized actions even if the program is compromised.

Working with AppArmor:


                        # Check AppArmor status
                        sudo aa-status

                        # View loaded profiles
                        sudo apparmor_status

                        # Set profile to complain mode (logging only)
                        sudo aa-complain /etc/apparmor.d/usr.bin.firefox

                        # Set profile to enforce mode
                        sudo aa-enforce /etc/apparmor.d/usr.bin.firefox

                        # Disable a profile
                        sudo ln -s /etc/apparmor.d/usr.bin.firefox /etc/apparmor.d/disable/
                        sudo apparmor_parser -R /etc/apparmor.d/usr.bin.firefox

Why does it matter?

Limits Damage from Exploits: If an application like a web browser is compromised, AppArmor prevents attackers from accessing sensitive files or escalating privileges. For example, a compromised Firefox process can't read your SSH keys or modify system files.
Default Protection: Many distributions (Ubuntu, SUSE, Arch via AUR) ship with AppArmor profiles for common services like Apache, MySQL, and Firefox, providing immediate protection.
Complements Firewalls: While firewalls control network traffic, AppArmor controls what applications can do on the system itself, creating defense in depth.

Why Linux is Inherently More Secure

1. Permission Model: Linux uses a strict user/group/other permission system with the principle of least privilege. Regular users cannot modify system files or install software system-wide without explicit sudo/root access.
2. Package Managers: Software is installed through cryptographically signed packages from trusted repositories (pacman, apt, dnf). This prevents malware disguised as legitimate software.
3. Open Source: The source code is publicly available and audited by thousands of developers worldwide, making it harder for backdoors or vulnerabilities to remain hidden.
4. No Admin by Default: Unlike Windows where users often run with administrator privileges, Linux requires explicit elevation (sudo) for privileged operations.
5. Smaller Attack Surface: You install only what you need. Minimal installations don't include unnecessary services that could be exploited.
6. Frequent Updates: Rolling release distributions (like Arch) receive security patches immediately, often within hours of disclosure.
